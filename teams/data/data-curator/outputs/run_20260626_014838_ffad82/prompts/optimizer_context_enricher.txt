## Methodology Text
This procedure has three functional layers (stages). Layer 1 classifies candidate posts for UMC relevance and dimension (unit of analysis: post). The vertical flow is: Platform records -> Rule + keyword screen -> Candidate posts -> Relevance coding -> Dimension tagging -> Classified post base (emphasis box).

Layer 2 aggregates classified posts at the district x dimension level and combines an administrative prior with platform observations using an empirical Bayes (EB) shrinkage rate model (unit: district x dimension). Vertical flow: Classified post base -> Dong linkage -> Exposure denominator -> Observed post rate -> EB shrinkage -> Posterior rate -> High-z_shift review cells (emphasis box). An arrow labeled "classified base" runs from Layer 1's Classified post base to the top of Layer 2.

Layer 3 is evidence-grounded judgment and runs five execution stages A-E (unit: post-level case). It is split by an INFORMATION BOUNDARY drawn as two dashed boxes:
- I_R: Stage A/B isolated inputs (restricted information set) -- Stage A post record branches to three isolated hypothesis agents: Abductive hyp., Forward hyp., Sequential hyp. (Stage B). Stage B sees only post text/metadata/dimension codes and does NOT see EB divergence or district profiles.
- I_J: Stage C/D/E evidence set (expanded judgment set) -- Stage C data specification -> Stage D rule-based evidence -> Stage E judgment -> Supported final coding.
- An arrow labeled "review cells" runs from Layer 2's High-z_shift review cells into Layer 3.
- Arrows run from the three hypothesis agents in I_R to Stage C in I_J (hypothesis hand-off crossing the information boundary).

Epistemic core: hypothesis generation (I_R, Stage A/B) cannot see EB divergence or district profiles, while only judgment (I_J, Stage C/D/E) sees expanded evidence. This information boundary blocks confirmation bias. High-divergence (high z_shift) cells trigger post-level hypotheses but do not determine the final judgment.
