# 스킬: journal-submission-checklist

## 사용 시점

선정·후보 저널이 정해진 뒤, 그 저널의 **공식 author guidelines**를 현재 원고 프로파일에 매핑해 투고 전 점검용 체크리스트를 만들거나 갱신해야 할 때. 1차 저널이 대안으로 바뀌거나(예: SSCR → ITD/GIQ) 재투고로 대상 저널이 달라지면 같은 절차를 반복한다.

저널 선정 자체(분위·적합도 비교)는 범위 밖이다 — 이미 선정된 저널의 투고요건 점검에만 쓴다.

## 목적

저널 공식 요건(분량·초록·키워드·인용양식·익명화·AI공개·필수진술·그림규격·투고시스템)을 원고의 실제 속성에 1:1 대조한 체크리스트를 산출해, 투고 전 형식 미달·데스크리젝트 위험을 사전에 드러낸다.

## 계약

- 읽는 입력: 선정 저널명, 저널 공식 author guidelines(1차 출처), 원고 프로파일 핸드오프(`.context/handoff/`).
- 만드는 출력: `.context/<저널약칭>-submission-checklist.md` 실파일(체크박스 형식), 미리알림 노트 annotate, 후속 에이전트(data-curator) 입력.
- 쓰면 안 되는 위치: 저널 선정 비교표(핸드오프 소관), 원고 본문, 공유 store 직접 편집.

## 입력

- 선정 저널명과 출판사
- 저널 공식 author guidelines URL(추측 금지 — 반드시 1차 출처 직접 조회)
- 원고 프로파일: 제목·핵심 기여·방법·키워드·인용 분포(핸드오프에서)
- 작업 방향(예: "방법론 기여 중심")

## 절차

1. 핸드오프에서 원고 프로파일과 선정 저널을 확인한다.
2. 저널 공식 author guidelines를 1차 출처(출판사 사이트)에서 직접 조회한다. 기억·추측으로 요건을 쓰지 않는다.
3. 요건을 영역별로 추출한다: 분량 한도, 초록 형식·단어수, 키워드 수, 인용양식(APA 등 판본), 익명화 수준(single/double-anonymized), AI/LLM 사용 공개, Title Page 필수항목, 필수 진술(윤리/IRB·동의·COI·Funding·Data availability), 그림·표 규격(dpi·번호·컬러), ORCID, 투고시스템·cover letter, preprint 정책.
4. 각 요건을 원고의 실제 속성에 대조해 `[ ]`/`[~]`/`[!]` 상태로 표시한다. 충족 가능·확인필요·위험을 구분한다.
5. 방법론 기여가 핵심이면 심사 대비 방어 항목(방법 타당성 근거 인용 매핑)을 별도 섹션으로 보존한다.
6. 충족 현황 요약표와 남은 위험·다음 단계를 적는다.
7. 산출물을 `.context/`에 실파일로 저장하고 경로를 통지한다(`team-inbox` 답장 + `reminders-team-bridge` annotate).
8. 검증 안 된 분위·IF·게재지·연도는 불확실로 표시한다.

## 출력 형식

- `.context/<저널약칭>-submission-checklist.md`: 영역별 헤더(A~) + 체크박스 + 충족 현황 요약표 + 남은 위험.
- 상태 표기: `[ ]` 미충족·미확인 / `[~]` 부분·원고측 확인필요 / `[!]` 위험·정밀확인.

## 내부 자원

- (보조 파일 없음 — 절차 단독)

## 품질 점검

- 모든 요건이 저널 공식 1차 출처에서 확인됐고 조회일자가 명시됐다.
- 각 요건이 원고 실제 속성에 대조됐다(막연한 일반론 아님).
- 데스크리젝트 직결 항목(분량 한도·AI 공개·익명화)이 위험 표시와 함께 드러났다.
- 산출물이 `.context/`에 실파일로 존재하고 경로가 통지됐다.
- 검증 안 된 수치는 불확실로 표시됐다.

## 자주 발생하는 실패 사례

- 노트에 '완료'로만 기록하고 실파일을 안 만듦 → 반드시 `.context/`에 실파일 생성 후 경로 통지.
- 요건을 기억·추측으로 작성 → 저널 공식 author guidelines 1차 조회 필수.
- 익명화 수준을 잘못 표기(single vs double) → 공식 문서에서 정확히 확인, 차이 시 정정 명시.
- 저널 선정 비교까지 끼워넣음 → 선정은 핸드오프 소관, 본 스킬은 점검만.
