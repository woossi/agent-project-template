# 스킬: fetch-paper-fulltext

## 사용 시점

학술 논문의 **본문(full text)/PDF를 실제로 읽어야** 할 때. 서지·abstract만으로는 답할 수 없는 검증, 구체적으로:

- 검증 질문이 논문의 **서술 방식·관행**(어떻게 쓰는가, 어디서 한정하는가, 몇 번 항변하는가)이거나, abstract에 안 나오는 **방법 절차·한계 서술·결과 단정 톤**을 확인해야 할 때.
- 인용 후보의 주장을 **본문 문장 단위로 대조**해야 할 때(차별화 대상·한계·정확 인용).
- 페이월·Cloudflare·기관인증으로 `WebFetch`가 막힌(402/403/리다이렉트) 논문 본문이 필요할 때.
- ★**전수 인용근거 추출 모드**: 새로 채택하는 **모든 신규 인용**에 대해, 그 인용이 우리 본문의 어느 논증·자리를 떠받치는지 **PDF 본문에서 정확한 인용 구절(quote)·페이지를 추출**해 manuscript-writer에 핸드오프해야 할 때(사용자 결정 2026-06-27: 인용은 초록이 아니라 정확한 본문 내용을 근거로 한다). paper-scout가 확보→추출까지 한 컨텍스트에서 연속 수행하며, **본문 삽입은 manuscript-writer 소관**(이 스킬은 검증된 근거 구절·위치·정합 코멘트 핸드오프까지).

서지정보(저자·연도·권호·DOI) 정확성만 필요하면 본문 확보 없이 Crossref로 충분하다(이 스킬 불요). 이 스킬은 **본문이 검증의 입력일 때만** 쓴다. 검색·서지대조 절차는 `scholarly-evidence-search`가 담당하고, 이 스킬은 그 절차의 "본문 확보" 단계를 깊이 다룬다.

## 목적

페이월·봇차단을 우회해 논문 본문을 합법 접근 경로 우선으로 확보하고, 그 본문에서 검증에 필요한 문장을 정확히 추출한다. 비용·차단·인증 제약을 계층적 폴백으로 처리한다.

## 계약

- 읽는 입력: 대상 논문의 DOI(우선) 또는 제목, 확인할 질문(서술방식/한계/방법/정확인용).
- 만드는 출력: 본문에서 추출한 **직접 인용 문장 + 위치 맥락**(검증 브리프의 근거로 사용). 본문 원문은 저장하지 않고 필요한 발췌만 남긴다.
- 쓰면 안 되는 위치: 저작권 본문 전문을 산출물·refs에 복제하지 않는다(필요 발췌·요약만). 기관 인증 쿠키를 파일/로그/inbox에 남기지 않는다(휘발적으로만 사용). 봇 우회는 AI봇 UA 위장 금지 규범을 지킨다(scrapling 기본 일반 브라우저 지문 사용).

## 입력

- **로컬 PDF 라이브러리(최우선).** `/Users/ujunbin/article/`(원문 PDF ~1063개, `저자_연도.pdf` 형식, Zotero·연세 기관 다운로드본). 작업경계에 **추가·승인됨**(agent-workspace.json defaults.allow + AGENTS.md; 읽기 전용, 2026-06-26 검증: Read 성공). `Read`로 직접 읽되 **PDF는 페이지 범위 필수**(`pages:"1-20"`, 한 요청 최대 20p) — 긴 논문은 목차·해당 절만 골라 읽는다. 페이월 우회보다 확실·합법·빠르나 커버리지가 부분적(우리 신규 ref 16건 중 Friston_2021만 매칭)이라 '있으면 최우선, 없으면 외부'.
- **WebFetch** — OA/공개 HTML 1차 시도(가장 가벼움).
- **scrapling MCP** — `mcp__scrapling__stealthy_fetch`(페이월·Cloudflare 우회: `solve_cloudflare=true`, `network_idle=true`), `mcp__scrapling__fetch`(저보호), `open_session`(쿠키 재사용). `cookies`/`session_id` 파라미터로 인증 세션 주입 가능. deferred면 `ToolSearch`로 스키마 로드.
- **Crossref** `api.crossref.org/works/<doi>` — DOI→OA 링크(`link`)·라이선스 확인.
- **기관 인증(Scopus 등)** — `scopus.com`(연세 계정 등)은 인증 필요. 에이전트가 직접 로그인 불가 → (a) 사용자가 로그인한 세션의 쿠키를 받아 scrapling `cookies`로 주입하거나, (b) 사용자에게 직접 검색·다운로드를 요청(폴백).

## 절차

1. **질문 유형 판별:** 검증이 *서술방식·관행·방법절차·정확인용*을 요구하면 본문 필수 → 이 스킬. 서지정확성만이면 중단(Crossref로 충분).
2. **로컬 PDF 라이브러리 우선:** 외부 fetch 전에 `/Users/ujunbin/article/`에 `저자_연도.pdf`가 있는지 먼저 본다(`ls /Users/ujunbin/article | grep -i <저자>`). 있으면 `Read`(`pages:` 범위 지정, 최대 20p/요청)로 직접 읽어 끝(가장 확실·합법·빠름). 없으면 외부로. (경계 승인됨 2026-06-26; 만약 다시 Read 차단되면 owner에 경계 확인 요청.)
3. **OA 우선:** 로컬에 없으면 Crossref `works/<doi>`의 `link`/라이선스로 OA 여부 확인. OA·공개 HTML이면 `WebFetch`로 1차 시도(doi.org는 cross-host 리다이렉트 빈번 → 리다이렉트 URL 재요청 또는 출판사 직링크). 페이월이라도 CC BY 등 OA면 **arXiv 저자본**(arxiv.org/html/<id>)이 봇친화적이라 가장 안정적.
4. **페이월·봇차단 우회:** WebFetch가 402/403/리다이렉트로 실패하면 `mcp__scrapling__stealthy_fetch(url, extraction_type='text', solve_cloudflare=true, network_idle=true, timeout≥60000)`. SAGE/Wiley/Springer 등 Cloudflare·인증벽은 이 경로로 다수 통과(실측: Synthese OA·SAGE OA 전문 취득).
5. **기관 인증(Scopus) 병행:** 위가 실패하는 진짜 페이월은 (a) **사용자 로그인 Scopus 세션 쿠키**를 받아 `stealthy_fetch(cookies=[...], url=풀텍스트)`로 주입 시도, 또는 (b) **사용자 폴백** — 사용자에게 "Scopus(연세 계정)에서 DOI <x> 검색·PDF 다운로드" 요청(세션 가이드의 `! <command>` 류 직접 수행). 쿠키는 휘발적으로만 사용하고 저장 금지.
6. **대용량 결과 분할 읽기:** scrapling 결과가 토큰 초과로 파일 저장되면, `grep`/`python`으로 검증 질문의 키워드 주변 문장만 추출(전문 재출력 금지).
7. **발췌·인계:** 검증 질문에 답하는 **직접 인용 문장 + 위치**만 브리프에 옮긴다. 본문 전문은 산출물에 복제하지 않는다. 확보 실패분은 `[본문 미확보]`로 표시하고 abstract 근거임을 명시.
8. **전수 인용근거 추출·핸드오프(전수 모드):** 새로 채택하는 인용 키마다 — (a) 그 인용이 **우리 본문의 어느 논증·자리**(섹션·주장)를 떠받치는지 먼저 명시하고, (b) 그 자리에 정확히 맞는 **본문 직접 인용 구절(verbatim quote)과 페이지/위치**를 PDF 본문에서 추출하며, (c) 그 구절이 우리 주장을 **정당화/차별화/한정** 중 무엇으로 떠받치는지 **정합 코멘트**를 붙인다. 억지로 맞는 구절을 만들지 않는다 — 본문에 우리 주장을 떠받치는 구절이 **없으면** `[정합 구절 없음 — 인용 부적합]`으로 정직 표시해 mw에 재검토 신호를 준다(과대인용 차단). 산출은 핸드오프 표(출력 형식)로 모아 mw에 전달하되, **본문 삽입은 하지 않는다**(mw 소관).

## 출력 형식

검증 브리프 안의 근거 항목: `저자(연도) — "직접 인용 문장" (위치/맥락) → 판정 근거`. 확보 경로(로컬PDF/OA/arXiv/scrapling/Scopus/사용자)를 1줄 남긴다. 미확보는 `[본문 미확보 — abstract 근거]`.

**전수 모드 핸드오프 표**(`.context/<주제>-citation-grounding.md`) — 인용 키마다 한 행:

| 인용 키 | 우리 본문 자리(섹션·주장) | 본문 직접 인용 구절(verbatim) | 페이지/위치 | 정합 유형(정당화/차별화/한정) | 확보 경로 |
|---|---|---|---|---|---|

정합 구절이 없으면 그 행에 `[정합 구절 없음 — 인용 부적합]`을, 본문 확보 자체가 실패하면 `[본문 미확보 — abstract 근거]`를 적어 mw에 정직 신호를 남긴다. 본문 전문은 표에 복제하지 않고 우리 주장 대조에 필요한 최소 구절만.

## 내부 자원

- (없음) — 외부 MCP(WebFetch·scrapling·Crossref) 도구에만 의존하며 보조 스크립트를 두지 않는다.

## 품질 점검

- 검증이 본문을 요구하는 유형인지 착수 전에 판별했는가(서지정확성만이면 이 스킬 불요).
- 합법 경로 우선순위(**로컬 PDF → OA/arXiv → scrapling → 기관인증/사용자 폴백**)를 지켰는가. 로컬 라이브러리를 외부 fetch보다 먼저 봤는가.
- 산출물에 본문 전문을 복제하지 않고 필요 발췌만 남겼는가.
- 인증 쿠키를 파일·로그·inbox에 남기지 않았는가.
- 확보 실패분을 `[본문 미확보]`로 정직히 표시했는가(abstract를 본문 확인인 척 쓰지 않음).
- (전수 모드) 인용 키마다 **우리 본문 자리 + verbatim 구절 + 페이지 + 정합 유형**을 핸드오프 표에 적었는가. 떠받치는 구절이 없으면 `[정합 구절 없음]`으로 정직 표시했는가(억지 구절 생성·과대인용 금지). 본문 삽입은 mw 소관으로 넘겼는가.

## 자주 발생하는 실패 사례

- **로컬 PDF 라이브러리를 안 보고 바로 외부 fetch** → `/Users/ujunbin/article/`에 원문이 있으면 페이월 우회·검색이 불필요. 외부 전에 `ls | grep -i <저자>` 먼저. (단 커버리지 부분적이라 없으면 외부로.)
- **로컬 PDF가 경계 밖이라 Read 차단** → guard Blocked. 임의 우회 금지 — 거버넌스 owner(data-curator)에 작업경계 추가 요청 후 사용(AGENTS.md allow + agent-workspace.json defaults.allow).
- **abstract로 '관행·서술방식' 질문에 답함** → 관행/서술은 본문 필수. 착수 시 질문 유형을 판별해 본문을 확보한다(이번 실패 교훈: abstract엔 'Discussion에 한계 1회 집중' 같은 결정 증거가 안 보임).
- **페이월에 WebFetch만 반복** → 402/403/리다이렉트면 즉시 scrapling `stealthy_fetch(solve_cloudflare=true)`로 전환. WebFetch 무한 재시도 금지.
- **Scopus를 에이전트가 직접 로그인 시도** → 불가(인증). 사용자 세션 쿠키 주입 또는 사용자 폴백으로 처리.
- **scrapling 결과 토큰 초과로 전문 재출력** → 파일 저장됨. grep/python으로 키워드 주변만 분할 추출.
- **본문 전문을 브리프·refs에 복제** → 저작권 위반 소지. 필요 발췌·요약만.
- **인증 쿠키를 산출물에 노출** → 금지. 휘발적으로만 사용.
- **(전수 모드) 본문에 없는 주장을 인용 구절로 만들어 붙임** → 과대인용·왜곡. 우리 자리를 떠받치는 구절이 본문에 없으면 `[정합 구절 없음 — 인용 부적합]`으로 정직 표시해 mw 재검토 신호를 준다(억지 동일시 금지).
- **(전수 모드) 본문 확보 실패를 정합 구절처럼 처리** → 확보 실패는 `[본문 미확보 — abstract 근거]`, 구절 부재는 `[정합 구절 없음]`으로 구분 표기. 두 경우 모두 mw가 인용 채택을 재판단하게 한다.
