# 학술 글 품질 검수 rubric (다차원 지표)

> 용도: `manuscript-review-gate` 절차 4단계(게이트 축별 판정)에서 **문체·수사 품질 차원**을 측정 가능한 지표로 판정할 때 참조한다.
> 출처: econ-writing-skill(Cochrane, McCloskey 35 rules, Head Introduction Formula, Shapiro Four Steps, Goldin&Katz, Kremer 등 50+ 정통 가이드 종합)의 review-checklist를 검수자 관점으로 내재화·조정. 외부 스킬에 런타임 의존하지 않는다.
> 성격: 이 rubric은 **판정 기준**이다. 집필은 하지 않으며, 차원별 플래그·점수와 재작업 지시만 산출한다.
> 팀 정책 조정: 외부 출처는 "hedge 부재"를 AI 문체로 보지만, 본 팀은 미검증분 hedge를 **의무**로 둔다. 따라서 본 rubric은 **방어적 과잉 hedge(감점)** 와 **정직한 불확실성 표시(가점)** 를 구분한다(아래 D2). 두 기준은 충돌이 아니라 균형이다.

## 판정 방식

각 차원을 0–2로 채점한다: **2=충족(PASS)** / **1=부분(PARTIAL, 권고)** / **0=미충족(FAIL, 반려)**.
차원별로 결함을 발견하면 `R<n>-F<k>`로 등록하고 파일·라인·재작업 지시를 단다. 점수는 게이트의 보조 신호이며, MAJOR 결함 1건이라도 있으면 해당 섹션은 PARTIAL 이하로 판정한다.

## A. 구조·수사 차원 (Field Expert + 수사 구조)

| ID | 지표 | 측정 기준 | 플래그(감점) |
|----|------|----------|-------------|
| A1 | 주장-우선 문단 | 문단이 인용·배경이 아니라 **주장으로 시작**하는가 | 문단 첫 문장이 `\citep`/"선행연구는"으로 시작하는 비율이 높음 |
| A2 | 기여 위치 | 핵심 기여·주요 결과가 서론 앞 3문단 내 명시되는가 | 기여가 뒤로 밀려 있음 |
| A3 | 문헌 공정성 | 문헌이 나열(laundry list)이 아니라 논증에 짜여 있고, **반대·불일치 연구도 인용**하는가 | 지지 문헌만 인용, 반대 견해 부재 |
| A4 | 결론 절제 | 결론이 새 결과·논증을 도입하지 않고 기여로 환원되는가 | 결론에 신규 주장 등장 |
| A5 | 상호참조 동기 | 섹션 전환이 기계적이지 않고 동기가 있는가 | "다음 절은…"식 기계 전환 남발 |

## B. 구체성·정량성 차원 (Writing Critic)

| ID | 지표 | 측정 기준 | 플래그(감점) |
|----|------|----------|-------------|
| B1 | 크기 명시 | 효과·격차를 **구체적 크기**로 쓰는가("약 2.5배" O / "상당한 효과" X) | "substantial/significant effect"류 모호 표현 |
| B2 | 통계 vs 실질 유의 구분 | 통계적 유의를 실질적(정책적) 유의와 혼동하지 않는가 | p값만으로 중요성 주장 |
| B3 | 능동태·직접성 | 수동·우회 표현("…인 것으로 나타났다"의 과용) 대신 직접 서술하는가 | 수동태·"It is shown" 과용 |
| B4 | throat-clearing 제거 | 문단 서두의 군더더기("…에 주목할 필요가 있다", "중요한 것은")가 없는가 | 서두 상투구 반복 |
| B5 | 표·그림 자족성 | 표·그림이 캡션·주석·단위만으로 단독 판독되는가 | 본문 없이는 표 해석 불가 |

## C. AI·상투 문체 탐지 차원

| ID | 지표 | 측정 기준 | 플래그(감점) |
|----|------|----------|-------------|
| C1 | 금지어 | "delve, crucial, landscape, multifaceted, notably, furthermore, comprehensive, robust(통계 외), utilize, leverage(동사), pivotal, groundbreaking, shed light on, pave the way" 및 한국어 대응 상투구("~을 조명한다", "다각적", "포괄적으로") 과용 없는가 | 금지어 빈출 |
| C2 | 문장 리듬 | 문장 길이가 단조롭지 않고 변주(대략 8–25어)되는가 | 모든 문장이 균일 길이 |
| C3 | 기계적 평행 | 모든 목록이 완벽 평행이 아니라 자연스러운가 | 과도하게 균일한 리스트 |
| C4 | 구체 제도 디테일 | 일반 플레이스홀더 대신 **해당 사례의 구체적 제도·맥락**이 있는가 | 추상적 일반론으로 대체 |

> 적용 주의: C 차원은 "기계가 쓴 듯한" 매끄러움을 감점하되, **정확한 학술 표현·정당한 전문용어는 감점하지 않는다**. 분야 published 논문 두 문단의 리듬을 기준으로 삼는다.

## D. 불확실성·정직성 차원 (팀 정책 결합)

| ID | 지표 | 측정 기준 | 판정 |
|----|------|----------|------|
| D1 | 미검증분 hedge **존재** | 검증되지 않은 주장이 명시적 hedge("잠정적으로", "may", "탐색적")로 표시되는가 | **부재 시 FAIL**(팀 정책 — 근거 없는 단정 금지) |
| D2 | 방어적 과잉 hedge **억제** | hedge가 **모든** 문장에 붙어 주장을 무력화하지 않는가. 검증된 사실은 단정하는가 | 과잉 시 PARTIAL(권고). 검증된 사실까지 hedge하면 감점 |
| D3 | 한계의 정직성 | 핵심 설계 약점(선택편향·대리지표·비식별성·검정력)이 한계절에 명시되는가 | 누락 시 FAIL |
| D4 | 과대주장 차단 | 탐색적 신호를 확증·유병률로 과확장하지 않는가 | 과확장 시 MAJOR 반려 |

> **D의 핵심**: "방어적 문체 지양"(D2)과 "미검증분 hedge 의무"(D1)는 **충돌이 아니라 짝**이다. 판정 규칙 = 검증된 사실은 단정(과잉 hedge면 D2 감점), 미검증분은 반드시 hedge(부재면 D1 FAIL). 즉 hedge의 **유무가 아니라 대상의 정합성**을 본다.

## 검수 산출 형식

체크리스트 재판정 R<n> 섹션에 문체 품질 소표를 추가한다:

```
### R<n> 문체 품질 (writing-quality-rubric)
| 차원 | 점수(0-2) | 플래그·근거(파일:라인) |
| A 구조·수사 | | |
| B 구체성·정량성 | | |
| C AI·상투 문체 | | |
| D 불확실성·정직성 | | |
→ 종합: <PASS|PARTIAL|FAIL>, 문체 결함 R<n>-F<k> 목록
```

각 0점(FAIL) 또는 MAJOR 플래그는 F-id로 승격해 재작업 지시(파일·라인·수정안 예시)를 단다. 적용은 manuscript-writer가 한다(검수자는 집필 금지).
