# 스킬: scholarly-evidence-search

## 사용 시점

논문화 작업에서 **동료평가(peer-reviewed) 문헌을 학술 검색 MCP로 찾아** 근거로 삼아야 할 때. 구체적으로:

- 우리 논문의 방법론 기여·실질 주장 각각을, 그것을 정당화/대비하는 **게재본 선행연구에 매핑**할 때(방어 체크리스트, refs 보강).
- 타깃 저널의 **적합도·분위(JCR Q)·게재 선례**를 동료평가 게재본으로 검증할 때.
- 심사자가 요구할 검증 항목(타당도·신뢰도·재현성, 한계·반론)을 **선행연구 근거로 묶어** 본문 갭을 점검할 때.

일반 웹 정보 조사가 아니라 **학술 근거가 필요할 때만** 쓴다. 저널 지표(IF/분위)의 최종 확정은 학술 MCP가 아니라 Clarivate JCR로 한다(이 스킬은 후보·추정·선례까지).

## 목적

학술 검색 MCP로 **동료평가 게재본만** 수집해, 우리 논문의 각 주장(기여·실질·메타틀)을 검증 가능한 선행연구에 결정적으로 연결한다. 근거의 신뢰도와 재사용성을 우선하며, 미검증·프리프린트·과대주장을 걸러낸다.

## 계약

- 읽는 입력: 우리 논문의 주장 목록(기여 축·실질 축·메타틀), 타깃 저널 후보, 검증할 심사 항목.
- 만드는 출력: `.context/`에 근거 매핑 브리프/체크리스트(주장 ↔ 게재본 ↔ 인용 링크). 본문/refs는 직접 수정하지 않고 보강 후보만 제시.
- 쓰면 안 되는 위치: 근거 미확정 상태로 본문(`research/UMC/parts/*`)·`refs.bib`를 고치지 않는다. 게재지 미확인 항목을 확정 인용으로 올리지 않는다.

## 입력

- 학술 검색 MCP 도구. **Consensus**(`mcp__claude_ai_Consensus__search`, claude.ai 커넥터 — 미연결 시 `/mcp`로 사용자 인증 필요) 또는 **Scholar Gateway**(`mcp__claude_ai_Scholar_Gateway__semanticSearch`, 인증 불필요. 단 결과가 커서 파일로 저장될 수 있음 — offset/limit로 분할 읽기). 도구가 deferred면 `ToolSearch`로 스키마를 먼저 로드한다.
- 우리 논문의 주장 목록과 타깃 저널.
- 저널 공식 author guidelines는 출판사 공식 페이지 1회 fetch(웹 검색 아님, 1차 출처).

## 절차

1. **주장 분해:** 우리 논문을 검증 단위로 쪼갠다(예: 기여 축마다, 실질 공백마다, 메타틀마다 1쿼리). 본문(`parts/*.tex`)에서 실제 방법론 정체성을 먼저 읽어 쿼리를 정확히 만든다.
2. **동료평가만 검색:** Consensus는 `exclude_preprints=true`로 호출. **필터는 사용자가 요청할 때만**(year/quartile 등) 건다. rate limit 회피를 위해 **한 번에 최대 3개 배치**, limit 시 ~30s 대기 후 재시도(같은 쿼리 무한 반복 금지).
3. **게재지 확인:** 결과의 저널명을 확인한다. `Unknown Journal`/미상은 **동료평가 여부 불확실**로 보고 채택 전 DOI로 검증. 저널 확인된 게재본을 우선 채택.
4. **주장↔근거 매핑:** 각 주장에 (a) 정당화 근거, (b) 가장 가까운 **차별화 대상 선행**(우리와 무엇이 다른지), (c) 정직하게 실어야 할 **한계·반론**을 함께 묶는다. 표준 도구(예: 공간 자기상관)는 기여 근거로 쓰지 않고 적용 도구로만 위치.
5. **저널 검증:** 타깃 저널에 우리 주장 두 축이 실제 **게재된 선례**를 찾아 적합도를 입증. 분위·IF는 추정으로 표기하고 **Clarivate JCR 재확인**을 단서로 남긴다.
6. **산출·인계:** `.context/`에 브리프/체크리스트로 정리(인용은 도구가 준 URL 그대로). refs 보강 후보·본문 갭·재확인 항목을 다음 단계로 명시.
7. 미리알림(umc) 태스크에 요약 체크백한다.

## 출력 형식

`.context/<주제>-brief.md` 또는 `-checklist.md`. 각 주장에 대해: 주장 → 게재본 근거(저자·연도·**저널명**·인용 링크) → 차별화/한계 → 본문 반영 여부 체크박스. 미확정은 `불확실`/`게재지 확인 필요`로 표기.

## 내부 자원

- (없음) — 이 스킬은 외부 학술 MCP 도구에만 의존하며 보조 스크립트를 두지 않는다.

## 품질 점검

- 채택 근거가 모두 **동료평가**이고 게재지가 확인되었는가(미확인은 `확인 필요` 표기).
- 각 주장에 정당화뿐 아니라 **차별화 대상·한계**가 함께 묶였는가(과대주장 방지).
- 저널 분위/IF는 추정으로 표기하고 **Clarivate JCR 재확인** 단서를 남겼는가.
- 본문/`refs.bib`를 근거 미확정 상태로 고치지 않았는가.
- 인용 URL을 도구가 준 그대로 보존했는가.

## 자주 발생하는 실패 사례

- **프리프린트를 게재본처럼 인용** → Consensus `exclude_preprints=true`, 결과 저널명 확인, `Unknown`은 DOI 검증 후 채택.
- **rate limit 오류** → 배치 ≤3, ~30s 대기 후 재시도.
- **저널 지표를 학술 MCP 숫자로 확정** → 추정으로만 쓰고 Clarivate JCR로 확정.
- **표준 도구를 기여 근거로 사용** → 표준 도구(공간 자기상관 등)는 적용 도구로만, 기여는 신규 설계에 집중.
- **근거 없이 본문·refs 수정** → 이 스킬은 매핑·후보 제시까지. 확정 인용은 DOI 검증 후 별도 작업.
- **Scholar Gateway 결과가 토큰 초과로 파일 저장** → offset/limit 분할 읽기로 100% 읽은 뒤 요약.

## 계약 연계

- 스킬은 반복되는 작업 묶음을 하나의 포괄 이름으로 승격한 재사용 절차다.
- 작업은 이 스킬을 `사용할 스킬` 입력으로 참조한다. 절차를 복사하지 않는다.
- 서브에이전트는 작업 입력을 받아 필요한 스킬 능력을 사용한다.
- 현재 진행상황과 handoff는 스킬에 저장하지 않는다.
