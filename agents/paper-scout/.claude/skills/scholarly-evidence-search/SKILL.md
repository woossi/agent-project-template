# 스킬: scholarly-evidence-search

## 사용 시점

논문화 작업에서 **동료평가(peer-reviewed) 문헌을 학술 검색 MCP로 찾아** 근거로 삼아야 할 때. 구체적으로:

- 우리 논문의 방법론 기여·실질 주장 각각을, 그것을 정당화/대비하는 **게재본 선행연구에 매핑**할 때(방어 체크리스트, refs 보강).
- 타깃 저널의 **적합도·분위(JCR Q)·게재 선례**를 동료평가 게재본으로 검증할 때.
- 심사자가 요구할 검증 항목(타당도·신뢰도·재현성, 한계·반론)을 **선행연구 근거로 묶어** 본문 갭을 점검할 때.
- `refs.bib` 엔트리의 **서지정보 정확성(저자·연도·저널·권호·페이지·DOI)을 원출처와 대조 검증**할 때(빌드 통과는 키 존재만 증명, 서지 사실은 미보장 — 외부 대조 필요).
- 우리 설계·주장의 근거를 찾을 때 **우리 팀이 이미 정리한 내부 지식(연구 노트·문헌 노트·개념 노트)이 있는지 먼저 확인**해야 할 때. 내부 자산이 외부 동료평가보다 우리 설계에 더 직접적인 기원·정합 근거인 경우가 많다.

일반 웹 정보 조사가 아니라 **학술 근거가 필요할 때만** 쓴다. 저널 지표(IF/분위)의 최종 확정은 학술 MCP가 아니라 Clarivate JCR로 한다(이 스킬은 후보·추정·선례까지).

## 목적

학술 검색 MCP로 **동료평가 게재본만** 수집해, 우리 논문의 각 주장(기여·실질·메타틀)을 검증 가능한 선행연구에 결정적으로 연결한다. 근거의 신뢰도와 재사용성을 우선하며, 미검증·프리프린트·과대주장을 걸러낸다.

## 계약

- 읽는 입력: 우리 논문의 주장 목록(기여 축·실질 축·메타틀), 타깃 저널 후보, 검증할 심사 항목. 서지 대조 시 대상 `refs.bib` 키 목록·DOI.
- 만드는 출력: `.context/`에 근거 매핑 브리프/체크리스트(주장 ↔ 게재본 ↔ 인용 링크) 또는 서지 정오 대조표(키 ↔ 원출처 ↔ 판정). 본문/refs는 직접 수정하지 않고 보강·정정 후보만 제시(refs 수정은 manuscript-writer 소관).
- 쓰면 안 되는 위치: 근거 미확정 상태로 본문(`research/UMC/parts/*`)·`refs.bib`를 고치지 않는다. 게재지 미확인 항목을 확정 인용으로 올리지 않는다.

## 입력

- **내부 RAG 도구(우선).** 팀 연구 지식베이스 obsidian-vault MCP(`mcp__obsidian-vault__search_notes`로 검색, `read_multiple_notes`로 정독, `get_vault_stats`로 범위 파악). 우리 본문 설계의 기원·정합 근거를 외부보다 먼저 찾는다. 작업 경계 트리 안의 RAG/지식 인프라(예: `project/umc/.../knowledge`)도 함께 점검. 도구가 deferred면 `ToolSearch`로 스키마를 먼저 로드한다.
- **기존 ref 자산.** 작업 트리의 `refs.bib`(이미 채택된 키 — 중복 회피 기준)와 이전에 정리된 신규 ref 관리 산출물(예: `parts/_newrefs_map.md`, `citekey_map.md`). 검증된 서지·인용 배치가 여기 있으면 재검색보다 재사용이 정확하고 빠르다.
- 학술 검색 MCP 도구(키워드+의미 **두 엔진 교차** 권장 — 서로 다른 논문을 surface한다). **Consensus**(`mcp__claude_ai_Consensus__search`, 키워드 매칭, claude.ai 커넥터 — 미연결 시 `/mcp`로 사용자 인증 필요) **와** **Scholar Gateway**(`mcp__claude_ai_Scholar_Gateway__semanticSearch`, 의미 기반. 인증 불필요. 단 결과가 커서 파일로 저장될 수 있음 — offset/limit로 분할 읽기). 도구가 deferred면 `ToolSearch`로 스키마를 먼저 로드한다.
- **인용 네트워크(snowballing) 채널.** 핵심 논문의 후방(참고문헌)·전방(피인용) 추적으로 정전(canonical)을 잡는다. Crossref `works/<doi>`의 `reference`(후방), Consensus/Gateway 결과의 인용수, 학술 페이지의 "cited by"를 활용.
- 우리 논문의 주장 목록과 타깃 저널.
- 저널 공식 author guidelines는 출판사 공식 페이지 1회 fetch(웹 검색 아님, 1차 출처).

## 절차

1. **주장 분해 + 본문 갭 즉시 진단:** 우리 논문을 검증 단위로 쪼갠다(예: 기여 축마다, 실질 공백마다, 메타틀마다 1쿼리). 본문(`parts/*.tex`)에서 실제 방법론 정체성을 먼저 읽되, **그 첫 읽기에서 인용이 빈약한 지점(예: 핵심 설계가 단일 인용 또는 무인용으로 정당화됨)을 명시적으로 진단**해 가장 큰 보강 대상으로 표시하고, 쿼리를 그 갭에 맞춰 설계한다.
2. **내부 자산 우선 조회:** 외부 검색보다 **먼저** 두 내부 소스를 본다. (a) **기존 ref 자산** — `refs.bib`(중복 회피 기준)와 이전 신규 ref 관리 산출물(`parts/_newrefs_map.md`·`citekey_map.md`). 이미 검증·배치된 서지가 갭을 메우면 재검색하지 않고 재사용한다. (b) **내부 RAG** — obsidian-vault `search_notes`를 우리 설계·차원·방법 키워드로 한·영 병행 조회하고 직결 노트는 `read_multiple_notes`로 정독. 본문 설계의 **기원·계보**와 **이미 정리된 대안·한계**가 내부에 있으면 외부보다 직접적인 근거다(단, 내부 노트·자산도 1차 출처가 아니면 §4 외부 대조 대상 — 내부 노트 서지는 오류 가능). 작업 경계 트리 안의 RAG/지식 인프라도 함께 점검.
3. **동료평가 외부 검색(두 엔진 교차):** 내부가 못 메운 갭을 외부로 채운다. 갭마다 **Consensus(키워드)와 Scholar Gateway(의미)를 교차** 호출한다 — 두 엔진은 서로 다른 논문을 surface하므로 한쪽만 쓰면 정전을 놓친다. Consensus는 `exclude_preprints=true`로 호출. **필터는 사용자가 요청할 때만**(year/quartile 등) 건다. rate limit 회피를 위해 **한 번에 최대 3개 배치**, 배치 사이에 간격을 두고, limit 시 ~30s 대기 후 재시도(같은 쿼리 무한 반복 금지).
3b. **인용 네트워크 추적(snowballing):** 가장 중요한 논문은 키워드가 아니라 인용 사슬로 잡힌다. 각 갭의 핵심 논문에 대해 **후방**(Crossref `works/<doi>`의 `reference` 배열 = 그 논문이 딛고 선 토대)과 **전방**(피인용 수가 큰 후속·"cited by")을 1회 추적해, 반복 등장하는 고인용 정전(canonical/landmark)을 채택 후보에 추가한다. 키워드 검색이 우연히 누락한 표준 출처를 메우는 단계다(과확장 금지 — 갭에 직결되는 것만).
4. **게재지·서지 확인:** 결과의 저널명을 확인한다. `Unknown Journal`/미상은 **동료평가 여부 불확실**로 보고 채택 전 DOI로 검증. 저널 확인된 게재본을 우선 채택. **DOI 기반 서지 대조는 Crossref REST API**(`api.crossref.org/works/<doi>`, 단일호스트라 WebFetch가 cross-host 리다이렉트 없이 1차 출처 JSON을 직접 반환 — doi.org 직접 fetch는 리다이렉트로 실패하기 쉬움). **online-first vs print 연도 차이**는 CSL-JSON transform(`.../transform/application/vnd.citationstyles.csl+json`)의 `published-print`/`published-online`로 분리 확인하며, **인용 키 연도는 정식 print 게재호 기준**으로 맞춘다(Crossref 단순 `issued`는 online-first 연도라 키와 달라 보여도 오류 아님). DOI 없는 챕터·미래(미래시점) 게재본은 출판사 공식 페이지 등 **독립 2차 출처로 교차확인**. 내부 RAG 노트에 적힌 DOI·서지도 같은 방식으로 1차 대조한 뒤 채택.
5. **주장↔근거 매핑:** 각 주장에 (a) 정당화 근거, (b) 가장 가까운 **차별화 대상 선행**(우리와 무엇이 다른지), (c) 정직하게 실어야 할 **한계·반론**을 함께 묶는다. 내부 RAG에서 나온 근거와 외부 게재본이 같은 주장을 떠받치면 **내부(기원·계보)와 외부(동료평가 정당화)를 함께** 묶는다. ★검증이 abstract로 안 되는 유형(논문의 **서술방식·관행·방법절차·정확 인용** 대조)이면 `fetch-paper-fulltext` 스킬로 본문을 확보해 문장 단위로 대조한다(페이월은 scrapling stealthy_fetch). 표준 도구(예: 공간 자기상관)는 기여 근거로 쓰지 않고 적용 도구로만 위치.
6. **저널 검증:** 타깃 저널에 우리 주장 두 축이 실제 **게재된 선례**를 찾아 적합도를 입증. 분위·IF는 추정으로 표기하고 **Clarivate JCR 재확인**을 단서로 남긴다.
7. **산출·인계:** `.context/`에 브리프/체크리스트로 정리(인용은 도구가 준 URL 그대로). refs 보강 후보·본문 갭·재확인 항목을 다음 단계로 명시.
8. 미리알림(umc) 태스크에 요약 체크백한다.

## 출력 형식

`.context/<주제>-brief.md` 또는 `-checklist.md`. 각 주장에 대해: 주장 → 게재본 근거(저자·연도·**저널명**·인용 링크) → 차별화/한계 → 본문 반영 여부 체크박스. 미확정은 `불확실`/`게재지 확인 필요`로 표기.

## 내부 자원

- (없음) — 이 스킬은 외부 학술 MCP·내부 RAG(obsidian-vault) MCP 도구에만 의존하며 보조 스크립트를 두지 않는다.

## 품질 점검

- **외부 검색 전에 내부 자산(기존 `refs.bib`·`_newrefs_map.md` + obsidian-vault RAG)을 먼저 조회**했는가(검증된 서지 재사용, 설계 기원·계보를 빠뜨리지 않음).
- ★**채택 후보를 '신규'로 표기·등록 권고하기 직전에, 후보 키 전수를 `refs.bib`에 grep으로 재선조회**했는가. 외부 검색을 건너뛴 케이스(이미 brief·노트에 정리된 키를 신규로 가정)에서도 이 전수 grep은 별도 강제다. 진입점 선조회만으로 '미존재'를 단정하지 않는다(직전 실수: 서론 탐색 때 채택 직전 grep 누락→3키 '미존재' 오판). 유사키·표기변형(저자 다른 철자·연도 print/online)까지 grep 패턴에 포함했는가.
- 핵심 갭마다 **두 엔진(Consensus 키워드 + Scholar Gateway 의미)을 교차**하고, 핵심 논문에 **인용 네트워크(후방 reference·전방 피인용)를 추적**해 정전을 확인했는가.
- 채택 근거가 모두 **동료평가**이고 게재지가 확인되었는가(미확인은 `확인 필요` 표기).
- 각 주장에 정당화뿐 아니라 **차별화 대상·한계**가 함께 묶였는가(과대주장 방지).
- 저널 분위/IF는 추정으로 표기하고 **Clarivate JCR 재확인** 단서를 남겼는가.
- 본문/`refs.bib`를 근거 미확정 상태로 고치지 않았는가.
- 인용 URL을 도구가 준 그대로 보존했는가.
- 서지 대조 시 DOI는 Crossref로 1차 출처 대조하고, online/print 연도 차이를 `published-print`로 구분해 키 연도를 print 기준으로 확정했는가(내부 노트에 적힌 서지도 동일 대조).

## 자주 발생하는 실패 사례

- **내부 자산을 건너뛰고 외부만 검색** → 설계 기원이 내부 노트에, 검증된 서지가 기존 `refs.bib`/`_newrefs_map.md`에 있는데 놓치고 중복 검색. 외부 **전에** 내부 자산(기존 ref + obsidian RAG 한·영 병행)을 먼저 돌린다.
- **채택 직전 전수 grep 누락→'미존재' 오판** → 진입 시 선조회를 했어도, brief·노트에 이미 정리된 키를 '신규'로 가정해 채택 권고 직전 재선조회를 빠뜨리면 이미 등록된 키를 '미존재'로 오판한다(실제 발생: 서론 7키 중 3키 오판). **채택/등록 권고 직전에 후보 키 전수를 다시 grep**하고, 유사키·표기변형까지 패턴에 넣는다.
- **키워드 한 엔진만으로 정전을 놓침** → 키워드 검색은 고인용 표준 출처를 우연히 누락한다. 두 엔진 교차 + 핵심 논문의 인용 네트워크(snowball) 추적으로 정전을 확보한다.
- **프리프린트를 게재본처럼 인용** → Consensus `exclude_preprints=true`, 결과 저널명 확인, `Unknown`은 DOI 검증 후 채택.
- **rate limit 오류** → 배치 ≤3, ~30s 대기 후 재시도.
- **저널 지표를 학술 MCP 숫자로 확정** → 추정으로만 쓰고 Clarivate JCR로 확정.
- **표준 도구를 기여 근거로 사용** → 표준 도구(공간 자기상관 등)는 적용 도구로만, 기여는 신규 설계에 집중.
- **근거 없이 본문·refs 수정** → 이 스킬은 매핑·후보 제시까지. 확정 인용은 DOI 검증 후 별도 작업.
- **Scholar Gateway 결과가 토큰 초과로 파일 저장** → offset/limit 분할 읽기로 100% 읽은 뒤 요약.
- **doi.org 직접 fetch가 cross-host 리다이렉트로 실패** → `api.crossref.org/works/<doi>`(단일호스트)로 서지 JSON 직접 대조.
- **online-first 연도를 게재 연도로 오인** → Crossref 단순 `issued`/`published`는 online-first일 수 있음. CSL-JSON `published-print`로 정식 게재호 연도를 확인해 인용 키와 대조(키=print 기준이면 정상).

## 계약 연계

- 스킬은 반복되는 작업 묶음을 하나의 포괄 이름으로 승격한 재사용 절차다.
- 작업은 이 스킬을 `사용할 스킬` 입력으로 참조한다. 절차를 복사하지 않는다.
- 서브에이전트는 작업 입력을 받아 필요한 스킬 능력을 사용한다.
- 현재 진행상황과 handoff는 스킬에 저장하지 않는다.
