# 스킬: stat-claim-verification

## 사용 시점

원고(`*.tex`)의 통계 수치 주장을 산출 데이터(CSV/parquet/txt)로 **독립 재현**해 셀 단위로 대조·판정해야 할 때. 구체적으로:

- 집필자가 보고한 표·본문·초록·그림 캡션의 수치가 산출물과 일치하는지 검증할 때(stats-validator의 1차 업무).
- 검수 게이트(manuscript-review-gate)·집필 자기검증에서 "이 수치 어디서 나왔나"를 확정해야 할 때.
- 절사규칙(trim/ceil/floor)·반올림·임계값 카운트처럼 코드 의도와 산출값의 정합을 따져야 할 때.

inbox "적용·보존 완료" 보고는 판정 근거가 아니라 **검증 대상 가설**이다. 보고를 신뢰의 근거로 삼지 않고 산출물을 직접 재현한다.

## 목적

tex 수치 주장과 산출 데이터를 충돌 없이 독립 재현해 ✅정합 / ⚠️불일치 / ❓출처불확실 세 부류로 판정하고, 임의 보정 없이 결함을 한 라운드라도 일찍 드러낸다. 재현이 곧 답이며 코드 grep은 보조(로직 부재 확정용)다.

## 계약

- 읽는 입력: 검증 대상 원고 수치(파일·라인), 그 수치를 낳은 산출물(작업 경계 내 CSV/parquet/txt), 산출 코드(있으면 재현 대조용·로직 부재 확정용).
- 만드는 출력: 검증 리포트(셀 단위 대조표 + 3분류 판정 + 재현 코드/명령), 받은편지함 통지(집필자·검수자·orchestrator).
- 쓰면 안 되는 위치: 원고 파일(집필자 소관 — 수정안 예시만 제시, 직접 보정 금지). 형제 에이전트 폴더(`agents/<다른이름>/`). 산출 데이터 원본(읽기 전용).

## 입력

- 검증 대상 tex 라인과 주장 수치(예: `body_results.tex:117`의 5수치).
- 산출 파일 경로 — **부재로 단정하기 전에 인접 트리까지 탐색**한다(분석 repo vs 논문화 트리 혼동 빈발). 입력이 "정본"인지 "스냅숏"인지 확인.
- Python(표준 라이브러리 csv/math로 충분). 추가 데이터 없이 동일 셀 재계산이 원칙.

## 절차

1. **산출 파일 특정.** 대상 수치의 산출 파일을 찾는다. `project/umc/analysis/`만 grep해 '미발견'으로 단정하지 말 것 — `research/UMC/`(논문화 트리)·인접 디렉토리까지 본다. 정본/스냅숏/구버전 트리를 구분한다(`_data_registry.md` 참조).
2. **Python 독립 재현(재현-우선).** 산출 파일에서 직접 값을 재계산한다. 코드 grep은 보조다 — 이번 답이 grep으로 안 나와도 CSV 몇 줄 재현이 곧장 줄 수 있다. grep은 '산출 로직이 코드에 명시돼 있는가'(출처불확실 분리)를 확정하는 용도.
3. **tex 셀 단위 대조.** 재현값을 tex의 표 셀·본문 분해수치·초록·그림 캡션과 1:1 대조한다. 부분합이 총계와 어긋나면 그 부분합이 무엇의 하위집합인지 본문이 정의하는지 확인하고, 정의가 없으면 결함으로 기록한다.
4. **3분류 판정.** ✅정합(재현값=tex) / ⚠️불일치(재현값≠tex, 결함) / ❓출처불확실(값은 재현되나 산출 로직이 코드에 명시 없음 — 불일치가 아니라 추적성 결함으로 분리). **역산 확정 ≠ 코드 확정**: 수치가 재현돼도 코드에 산출로직이 없으면 ❓로 둔다.
5. **반올림·절사 함정 점검.** `round`/`ceil`/`floor`/`trunc`를 **전부 비교**하고 'Python `round` 기준' 전제를 명시한다(banker's rounding: Python `round(2.5)=2≠3`). 임계값 카운트는 추정 금지 — 임계값별 카운트(예: 54/22/3)로 확정한다. 인접 시나리오(55/21/4)와 비슷해 추정하면 틀린다.
6. **통지.** 검증 리포트 경로와 3분류 결과를 받은편지함으로 집필자·검수자(필요 시 orchestrator)에 reply한다. 수치 결함은 임의 보정 금지 — 실측 확정 후 적용을 지정한다.

## 출력 형식

- 검증 리포트(`.context/.../NN_*_verification.md`):
  - 셀 단위 대조표 `| tex 위치 | 주장값 | 재현값 | 판정(✅/⚠️/❓) | 재현 근거 |`
  - 재현 코드/명령(동일 셀 재계산, 표준 라이브러리)
  - 종합: ✅정합 / ⚠️불일치(결함·재작업 지시) / ❓출처불확실(추적성 보강 권고) 목록
- 받은편지함 reply 1건(리포트 경로 + 3분류 요약 + 잔존 [불확실]).

## 내부 자원

(없음 — 절차 전용 스킬. 재현은 대상별 임시 Python으로 수행하고 리포트는 호출자 `.context/`에 둔다.)

## 품질 점검

- 판정은 inbox 보고가 아니라 **산출물 직접 재현**에 근거한다.
- ✅/⚠️/❓ 3분류가 모든 대상 수치에 빠짐없이 부여된다.
- 반올림 의존 수치는 round/ceil/floor/trunc 비교 결과와 전제 가정이 명시된다.
- ⚠️불일치는 임의 보정 없이 실측 확정 연계로 라우팅된다.
- 산출 파일 '부재' 판정 전에 논문화 트리·인접 디렉토리 탐색이 선행됐다.

## 자주 발생하는 실패 사례

- **`project/umc/analysis/`만 grep해 '미발견→불확실'로 단정** → 산출물이 논문화 트리(`research/UMC/active_inference/reproduction/` 등)에 있을 수 있다. 인접 트리까지 탐색 후 판정.
- **banker's rounding 함정** → Python `round(2.5)=2≠3`. round/ceil/floor/trunc 전부 비교하고 기준 전제를 명시.
- **baseline 시나리오 추정** → 임계값 카운트(54/22/3)로 확정. 인접값과 비슷해 추정하면 틀린다.
- **역산 확정을 코드 확정으로 오기** → '수치는 재현되나 코드에 산출로직 명시 없음'은 불일치 아님/출처불확실(❓)로 분리.
- **코드 grep을 1차 수단으로 의존** → 재현-우선. grep은 로직 부재 확정용 보조.
