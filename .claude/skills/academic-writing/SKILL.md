# 스킬: academic-writing

## 사용 시점

SSCI Q1 투고를 목표로 학술 논문의 한 섹션(서론·관련연구·방법·결과·논의) 또는 초록을 집필·재작성·정련할 때. 구체적으로:

- `section-writer` 서브에이전트가 배정된 섹션을 독립 컨텍스트에서 정밀 집필할 때.
- manuscript-writer가 섹션 단위 집필을 위임하거나, 기여 명제·hedging·인용 정합을 점검할 때.
- 투고 직전 분량(SSCR 10,000단어)·초록(150-200단어)·APA7 규격에 원고를 정렬할 때.

이미 완결도 높은 초고가 있으면 "백지 집필"이 아니라 이 절차의 **점검·정렬 모드**로 적용한다(섹션 move 누락 보강, 과대지지 인용 교정, hedging 정합).

## 목적

섹션별 수사 구조(rhetorical moves)·기여 명제·근거 무결성·투고 규격을 한 절차로 통합해, 즉흥적 집필이 아니라 SSCI Q1 심사기준에 정렬된 검증 가능한 원고를 산출한다. 수치·주장은 데이터·출처로 뒷받침하고, 미검증분은 불확실로 표시한다.

## 계약

- 읽는 입력: 배정 섹션의 현 초고(`parts/*.tex` 등), 기여 명제, stats-validator 수치검증 결과, 투고 체크리스트(paper-scout), `references/` 내부 자원.
- 만드는 출력: 해당 섹션의 집필·재작성 텍스트(원고 트리 반영), 변경 근거 메모(`.context/`).
- 쓰면 안 되는 위치: 작업 진행상황·handoff를 이 스킬에 저장하지 않는다(→ `.context/`). 도메인 수치 사실을 스킬에 박지 않는다.

## 입력

- 배정 섹션 식별자(서론 / 관련연구 / 방법 / 결과 / 논의 / 초록)와 대상 파일 경로
- 논문의 기여 명제 2축(이 프로젝트: EB 수축 비교 측정-기제 이행 / 정보차단 멀티에이전트 역행추론) — 또는 해당 논문의 기여
- 보존 필수 수치·통계량 목록(창작 금지)
- 투고 규격: 분량 한도, 초록 길이, 인용 스타일(기본 APA7), 익명화 요건
- 검증 자원: stats-validator 검증 리포트(수치), paper-scout 체크리스트(양식)

## 절차

1. **모드 판정.** 현 초고를 정독하고 "백지 집필"인지 "점검·정렬"인지 판정한다. 완결도 높은 초고면 전면 재작성을 피하고 move 누락·정합 결함만 보강한다.
2. **섹션 수사 구조 적용.** `references/section-moves.md`에서 배정 섹션의 전형적 move 구조(서론 CARS, 방법 재현가능성, 결과 수치 서술, 논의 해석-한계-함의)를 읽고, 현 텍스트에 누락된 move를 식별·보강한다.
3. **기여 명제 정렬.** `references/contribution-frame.md`의 gap→contribution→claim 프레임으로, 섹션이 기여 2축을 일관되게 지지하는지 확인한다. 서론의 기여 선언 ↔ 방법의 구현 ↔ 결과의 증거 ↔ 논의의 함의가 서로 모순되지 않게 한다.
4. **근거 무결성 점검.** `references/citation-evidence.md` 규칙으로 모든 주장-출처 정합을 검증한다. 출처가 주장을 실제로 지지하는지(과대지지 금지), 인용키가 서지에 존재하는지 확인한다. 새 수치·통계량을 창작하지 않는다 — 보존 목록과 stats-validator 검증분만 단정하고, 미검증은 hedging한다.
5. **hedging·불확실성 표기.** `references/hedging-rules.md`에 따라 확정/잠정 주장을 구분한다. 검증된 사실은 단정, 미검증·추정은 명시적 hedge("suggests", "may", "잠정적으로", "불확실"), 한계는 논의에 명시.
6. **투고 규격 정렬.** `references/submission-specs.md`로 분량·초록 길이·APA7·익명화를 점검한다. 분량 초과 위험 시 보조분석 supplementary 이관을 제안한다(직접 삭제는 보고 후).
7. **검증.** 인용키 무결성(undefined citation 0), \ref↔\label 정합을 확인한다. ★섹션 **레벨**을 바꿀 때(\section↔\subsection 승강)는 정합이 깨지지 않아도 의미가 틀어질 수 있다 — 변경 대상 라벨이 외부 파일에서 ``§N장''/``N절''로 참조되는지 먼저 `grep`하고, 참조 단위(장/절)와 라벨의 레벨을 일치시켜 유지한다(예: 외부가 ``\ref{sec:x}장''으로 쓰면 `sec:x`는 \section에 둔다). LaTeX이면 scratchpad에서 빌드해 컴파일 통과를 확인하고 원본 트리를 오염시키지 않는다.
8. **변경 근거 기록.** 무엇을 왜 바꿨는지, 어떤 주장이 미검증(불확실)인지 `.context/`에 남기고, 게이트 검수가 필요하면 quality-reviewer에 통지한다.

## 출력 형식

- 배정 섹션의 집필·재작성 텍스트(원고 파일 반영)
- 변경 근거·미검증 항목 메모(`.context/`)
- 보존 수치 무손상, 인용 무결성 통과, (LaTeX) 컴파일 통과 증빙

## 내부 자원

- `references/` — 섹션 집필 참조 자원 폴더
- `references/section-moves.md` — IMRaD 섹션별 전형적 수사 move 구조(서론 CARS, 방법, 결과, 논의)
- `references/contribution-frame.md` — gap→contribution→claim 기여 명제 프레임워크
- `references/citation-evidence.md` — 인용·근거 무결성 규칙(주장-출처 정합, 과대지지 금지)
- `references/hedging-rules.md` — hedging·불확실성 표기 규칙
- `references/submission-specs.md` — APA7·분량(SSCR 10,000단어)·초록(150-200단어)·익명화 규격

## 품질 점검

- 보존 필수 수치·통계량이 창작·변형 없이 유지되는가.
- 모든 인용이 서지에 존재하고 출처가 주장을 과대지지하지 않는가(undefined citation 0).
- 미검증 주장이 hedging으로 표시되었는가(단정 금지).
- 배정 섹션의 전형적 move가 누락 없이 존재하는가.
- 투고 규격(분량·초록·APA7·익명화) 위반이 없는가.
- (LaTeX) scratchpad 빌드가 exit 0이고 원본 트리를 오염시키지 않았는가.

## 자주 발생하는 실패 사례

- 완결 초고를 백지 재작성 → 모드 판정 후 점검·정렬로 전환한다.
- 수치·통계량 창작 → 보존 목록·stats-validator 검증분만 단정하고 나머지는 hedge한다.
- 인용 과대지지(출처가 주장만큼 강하게 말하지 않음) → `references/citation-evidence.md`로 주장 강도를 출처에 맞춰 낮춘다.
- 분량 초과를 본문에서 임의 삭제 → supplementary 이관을 제안하고 보고 후 결정한다.
- 진행 로그를 스킬에 기록 → `.context/`에 둔다.
