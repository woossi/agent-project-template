# IMRaD 섹션별 수사 move 구조

각 섹션이 따라야 하는 전형적 move(수사 단계)와 문장 수준 규칙. 톱저널 관행을 사회과학 SSCI Q1(1차 저널 SSCR/SAGE) 맥락에 맞춰 정리한다. 백지 집필이 아니라 **점검·정렬 모드**에서는 현 텍스트에 누락된 move만 식별·보강한다.

> 근거: Wikipedia "Signs of AI writing"(WikiProject AI Cleanup), Limdongcheng/cc-academic-writing `references/research-paper.md`(Science/Nature급 move 구조). 사회과학 관행(CARS 모델, Swales 1990; APA 7th)에 맞춰 조정.

---

## 서론(Introduction)

전형적 move 순서 — Swales CARS(Create A Research Space) 모델 기반:

1. **영역 확립(Establishing a territory)** — 연구 분야의 중요성을 간결히. 과장(significance inflation) 금지.
2. **문헌 지형(Literature landscape)** — 연대순이 아니라 **지적 계보·주제별**로 조직. "A는 X를 보고했다. B는 Y를 발견했다" 식 나열(약한 서술) 금지 → 이해의 진화를 추적.
3. **간극 진술(Gap statement)** — 무엇이 미해결인지. 언급한 **모든 선행 접근을 배제(compound exclusion)**해야 함. 선행연구 폄하 금지 — 그들이 직면하지 않은 제약을 다룬다는 틀로 기여를 제시.
4. **본 연구 접근(Your approach)** — 한 문장. **결과(Results) 섹션의 언어와 일치**시킴.
5. **혁신 틀(Innovation framing)** — 율속 단계(rate-limiting step)의 이동 또는 새로 다뤄진 제약. exclusion pattern으로.
6. **닫는 문장** — 성취 나열이 아니라 **독립적 과학 명제**(standalone principle)로.

문장 수준 규칙:
- 간극 진술은 앞서 언급한 선행 접근을 빠짐없이 배제해야 한다.
- 선행연구를 깎아내리지 말고, 그들이 마주하지 않은 조건을 다룬다고 틀 짓는다.
- 분량: 사회과학 논문 기준 대략 전체의 15-20%. 밀도 있되 산만하지 않게.

이 프로젝트 적용: 기여 2축(EB 수축 비교 / 정보차단 멀티에이전트 역행추론)을 §1.4 기여 선언에 exclusion pattern으로 명시하고, 방법·결과·논의 언어와 일치시킨다.

## 관련연구(Related Work / 이론적 배경)

사회과학 논문에서 별도 섹션으로 두는 경우. move:

1. **이론적 좌표(Theoretical positioning)** — 본 연구를 어떤 이론 전통에 접속하는지. 차용(절차적 발견법)과 동일성 주장을 구분.
2. **경쟁 관점 대비(Competing perspectives)** — 대립하는 접근의 좌표를 그린다(예: place-based vs people-based 정책).
3. **간극으로의 수렴(Convergence to gap)** — 문헌이 본 연구의 간극으로 수렴함을 보인다.

규칙: 인용은 주제별로 묶고, 각 인용이 **주장을 실제로 지지**하는지 확인(과대지지 금지, `citation-evidence.md`).

## 방법(Methods)

전형적 move 순서:

1. **재료·데이터(Materials/Data)** — 출처, 표본, 기간, 규모.
2. **장비·도구(Instruments/Tools)** — 모델, 핵심 설정. (계산연구면 소프트웨어·알고리즘·하이퍼파라미터.)
3. **조건(Conditions)** — 처리, 표본 그룹, 무엇이 구조적으로 다르고 왜.
4. **데이터 분석(Data analysis)** — 통계 방법, 적합, 추정.
5. **그림-방법 대응(Figure-method correspondence)** — 데이터 패널마다 여기 기술된 방법으로 추적 가능해야 함.
6. **하위 실험별 조직** — 연속 서사가 아니라 이산 하위 섹션(전처리, 측정, 추정, 검증 등).

문장 수준 규칙:
- **재현가능성 표준**: "같은 분야의 유능한 연구자가 이 섹션만으로 재현할 수 있어야 한다."
- 데이터를 보이는 모든 그림은 기술된 방법에 연결되어야 한다.
- 분량: 본문에서 압축하고 세부는 supplementary로 이관 가능. 단, 완결성 표준은 동일.

이 프로젝트 적용: EB 수축, 공간적 자기상관/LISA, 텍스트 전처리·LLM 코딩(131,792건), 정보차단 멀티에이전트 절차(I_R 제한정보집합)를 각 하위 섹션으로. stats-validator 검증 결과와 일치시킨다.

## 결과(Results)

전형적 move 순서:

1. **개관 진술(Overview)** — 1-2문장으로 분석이 드러낸 바를 요약.
2. **정확한 데이터 제시** — 그림·표·본문이 내적으로 일치.
3. **관찰 후 해석(Observation before interpretation)** — 본 것을 먼저 기술하고, 그다음 의미. 역순 금지.
4. **자연스러운 절 경계에서 그림 참조** — "(그림 3A)"는 절이 끊기는 곳에, 절 중간 금지.
5. **각 그림 섹션은 다리(bridge)로 마감** — 다음 그림이 답할 한계·질문을 제기.

문장 수준 규칙:
- 증거를 앞세우고 의미를 뒤따르게 한다.
- "한계 → 검증" 패턴: "데이터는 X가 작동함을 보이나 왜인지는 설명 못 한다. 기제를 분리하기 위해…"
- 수치 서술은 보존 필수 목록·stats-validator 검증분만 단정. 미검증은 hedging(`hedging-rules.md`).

분량: 그림 섹션당 한 단락 정도.

## 논의(Discussion)

전형적 move 순서:

1. **서론 간극에 답하기** — 앞서 제기한 간극 질문에 명시적으로 답한다.
2. **선행연구와 비교** — 일치하는가. 갈리면 왜인지 정직하게 설명.
3. **기저 기제 논의** — 결과 진술을 넘어선 인과·기제 추론.
4. **혁신 명시** — 분야가 어떻게 다르게 생각해야 하는가(성취가 아니라 **일반화 가능한 원리**).
5. **한계 인정** — 이 연구가 답할 수 없는 것. 심사자에게 과학적 성숙도를 신호.

문장 수준 규칙:
- 결과를 요약하지 않는다(결과 섹션과 역할이 다름).
- 불일치를 정직하게 틀 짓는다: "선행은 X를 가정했으나, 우리는 조건이 중요함을 발견."
- "발견한 것"과 "분야에 대한 함의"를 구분.

분량: 사회과학 기준 전체의 20-25%. (Results와 병합하는 저널이면 각 결과 하위 섹션 끝에 해석을 배치.)

## 초록(Abstract)

5문장 구조(SSCR 150-200단어):

| 요소 | 분량 | 기능 |
| --- | --- | --- |
| 배경·동기 | 1문장 | 분야가 왜 이 연구를 필요로 하는가 |
| 문제 | 1문장 | 구체적 간극·도전 |
| 방법·접근 | 1문장 | 최상위 수준에서 무엇을 했는가 |
| 핵심 결과 | 1-2문장 | 표제 수치·지표 |
| 의의 | 1문장 | 일반화 가능한 원리·설계 규칙 |

move 순서: 시급성 확립 → 간극 지목 → 해법 기술 → 성과 보고 → 원리 진술.

검증 테스트: "비전문가가 초록만 읽고 무엇을 발견했고 왜 중요하며 왜 새로운지 파악할 수 있는가?"
