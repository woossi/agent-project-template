# 스킬: register-term

## 사용 시점

프로젝트에서 쓰이는 용어·약어·번역어를 `.claude/memory/word.json` 사전에 추가하거나 갱신해야 할 때. 구체적으로:

- 사용자가 "이 용어를 등록/추가/정의해 달라"고 요청할 때.
- 대화나 코드에서 의미가 불명확한 약어·전문어가 반복 등장해, 합의된 정의를 사전에 남겨 두는 편이 나을 때(이 경우 등록 전에 사용자에게 정의를 확인한다).

일회성 설명으로 충분하고 재사용 가치가 없는 단어는 등록하지 않는다.

## 목적

용어의 정의를 사용자에게 명확히 받아, 필수 4개 필드(`term`/`ko`/`definition`/`use_when`)를 갖춘 항목으로 `word.json`에 안전하게(중복·필드 누락·JSON 깨짐 없이) 기록한다. 즉흥적이거나 불완전한 용어 등록을 방지한다.

## 입력

- 등록 대상 용어 1개에 대한 4개 필드:
  - `term` — 용어의 원어/표준 표기 (예: `RAG`, `LISA`)
  - `ko` — 한국어 표현 (예: `검색 증강 생성`)
  - `definition` — 한 문장 정의
  - `use_when` — 이 용어를 쓰는 맥락/조건
- 대상 파일: `.claude/memory/word.json` (기본값. 다르면 `--word-file`로 지정)

## 절차

1. **정의를 명확히 확보한다.** 위 4개 필드 중 사용자가 주지 않았거나 모호한 항목이 있으면, **추측하지 말고 사용자에게 직접 물어본다.** 4개가 모두 분명해질 때까지 등록하지 않는다.
2. **등록 전 사전 상태를 점검한다(선택):** `python scripts/register_term.py --check`
3. **용어를 추가한다:**
   ```bash
   python scripts/register_term.py add \
     --term "<원어>" --ko "<한국어>" \
     --definition "<정의>" --use-when "<사용 맥락>"
   ```
   - 동일 `term`이 이미 있으면 스크립트가 거부한다(종료코드 1). 의도적으로 갱신하려면 사용자 확인 후 `--allow-update`를 붙인다.
4. **검증한다:** `python scripts/register_term.py --check` 가 종료코드 0인지 확인한다.
5. 추가한 용어와 변경 파일을 사용자에게 보고한다.

## 출력 형식

`.claude/memory/word.json` 의 `terms` 배열에 항목 하나가 추가/갱신된다. 항목 형태:

```json
{
  "term": "RAG",
  "ko": "검색 증강 생성",
  "definition": "외부 지식을 검색해 LLM 응답에 결합하는 기법",
  "use_when": "모델 지식만으로 부족해 출처가 필요한 답변을 만들 때"
}
```

파일은 UTF-8·2칸 들여쓰기·끝에 개행으로 저장되며 한글은 그대로 보존된다.

## 내부 자원

- `scripts/register_term.py` — 용어 추가/검증 실행 코드. `add`(추가), `--check`(무결성 점검만), `--word-file`(대상 경로), `--allow-update`(동일 term 덮어쓰기) 지원. 필수 필드 검증·중복 term 검사(대소문자 무시)·JSON 유효성 검증을 수행하며, 사용자에게 직접 묻지는 않는다(묻기는 위 절차의 1단계에서 Claude가 한다).
- `references/notes.md` — 용어 등록 기준(무엇을 등록/제외할지), 각 필드 작성 지침, 자주 하는 실수 메모.

## 품질 점검

- 등록 후 `python scripts/register_term.py --check` 가 종료코드 0이어야 한다(JSON 유효·필수 4필드 충족·중복 term 없음).
- 추가된 항목의 4개 필드가 모두 비어 있지 않아야 한다.
- 같은 `term`(대소문자 무시)이 사전에 중복으로 존재하지 않아야 한다.

## 자주 발생하는 실패 사례

- **필드 누락으로 추가 거부(종료코드 1)** → 빠진 필드를 사용자에게 물어 4개를 모두 채운 뒤 다시 실행한다.
- **중복 term 으로 거부(종료코드 1)** → 새 용어가 아니라 기존 항목 수정이라면, 사용자에게 확인하고 `--allow-update` 로 갱신한다.
- **`word.json` 을 못 찾음** → 저장소 루트에서 실행하거나 `--word-file <경로>` 로 대상을 지정한다.
- **불명확한 정의를 임의로 채워 등록** → 금지. 모호하면 절차 1단계로 돌아가 사용자에게 정의를 확인한다.
