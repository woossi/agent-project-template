# 작업

agent-template 프로젝트의 현재 작업 패킷입니다. 작업은 진행 중인 작업 단위이며, 다른 에이전트가 목표·입력·출력·위험·완료 기준을 파악할 수 있을 만큼의 정보를 담습니다.

## 입력 규칙

다음 경우에 작업을 만들거나 수정합니다.

- 요청이 여러 단계로 이루어질 때
- 산출물을 구체적으로 확정해야 할 때
- 여러 컴포넌트가 변경될 수 있을 때
- 검증이 중요할 때
- 작업이 여러 세션에 걸칠 수 있을 때

## 출력 규칙

작업 패킷은 상태·목표·배경·입력·기대 산출물·절차·필요한 결정·위험·검증·완료 기준을 담습니다. 아래 양식을 사용하며, 지속적 메모리가 아닌 현재 작업만 여기에 둡니다.

## 작업 입력 양식

```md
# 작업

## 상태
대기 | 진행 중 | 막힘 | 완료

## 목표
한 문장으로 표현한 결과.

## 배경
이 작업이 필요한 이유.

## 입력
- 입력 파일, 데이터, 프롬프트 또는 출처

## 기대 산출물
- 구체적인 결과물

## 절차
1. 단계

## 필요한 결정
- 결정 사항 또는 질문

## 위험
- 위험 또는 불확실성

## 검증
- 출력이 유효함을 증명하는 점검

## 완료 기준
- 작업을 완료로 만드는 조건
```

## 현재 작업

상태: 대기

목표: (아직 작업이 없습니다. 첫 작업이 생기면 위 작업 입력 양식으로 채우세요.)

기대 산출물:

- (미정)

완료 기준:

- (미정)
